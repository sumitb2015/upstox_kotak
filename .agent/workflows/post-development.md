---
description: Post Development Workflow
---

1. Ensure that the code has followed the structure and the best practices, guidelines, testing and the api guide mentioned in the rules.
2. Ensure that no secret and passwords are hardcoded in the code
3. Ensure that all the code functions are properly documented
4. Ensure that the live.py function has a header documentation which provides a summary of the entry, exit, tsl,sl conditions on the strategic logic
5. Ensure that proper logging and exception handling has been taken care of
6. Clean up the debug code and test files which were prepated during the implementation
7. Always try to keep the overall folders and files clean based on the necessary files.
8. Check in the code to Git/GitHub, ensuring `.gitignore` is respected and no secrets are committed.
